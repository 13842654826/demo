🧪 自定义CSS干扰测试页

请开启/关闭ArkWeb夜间模式，观察下方区域的渲染优先级与算法容错。

1. 系统级暗色适配

跟随系统深色模式声明（@media prefers-color-scheme: dark）

若ArkWeb强制夜间模式，预期：应覆盖或融合系统媒体查询，而非出现双套背景色。

2. !important 强制暗色

站点使用 !important 锁死深色背景与浅色文字。

测试点：ArkWeb注入的暗压样式优先级是否高于 !important，或是否智能降级。

3. 站点已自带 invert(1)

页面元素已自行反色（常见于老旧阅读插件或第三方组件）。

测试点：开启夜间模式后，不应双重反色变回亮色。引擎应识别并跳过或应用 invert(0)。

4. filter: none 保护区域

明确声明 filter: none !important，模拟地图/视频/验证码等需保留原色的区域。

测试点：ArkWeb应尊重此声明，不对其施加全局反色/暗压滤镜。

5. color-scheme 声明冲突

only dark
only light

测试浏览器对 color-scheme 的解析策略：夜间模式下是否强制覆盖，或保持声明状态。
